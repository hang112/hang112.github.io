5.单例模式

防止滥用单例模式. 比如有一个Log类, 我们把他设计成了单例模式.起初这不是什么大问题. 我们记录单独的日志文件,所以只需要一个实例. 然后,随着开发周期的逐次循环,我们遇到了麻烦. 每个团队的成员都使用日志记录各自的诊断信息,大量的日志倾泻在文件里. 程序员需要翻过很多页才能找到他关心的记录.
我们想将日志分散到多个文件中解决这点.可是我们做不到,因为log类不允许我们创建多个实例. 为了让log类支持多个实例,我们需要修改类和提及它的每一行代码. 更糟的是, 假如你的Log类写在库中,给多个游戏提供支持.为了改变设计,需要在多组人之间协调,他们中的大多是既没有时间,也没有动机修复它.

直线的两种方程表示

1.斜截式

y = mx + b。

在有些情况下斜截式不是最好的表示方式，例如直线x = 0没法用斜截式表示，因为这条直线的m是无穷大。
因此有另一种隐含式。

2.隐含式

y - mx - b = 0。

m是斜率，b是穿过y轴时的y值。
在不改变方程的零点情况下，可以对隐式方程两边同乘以一个常量。对任意非零的k，kf(x, y) = 0 与 f(x, y) = 0是相同的曲线。所以，同一条直线可以有很多种不同的表示方式，例如： 

2y - 2mx - 2b = 0。

更通用的形式是 

f(x, y) = Ax + By + C = 0。

过两点($x_0$, $y_0$), ($x_1$, $y_1$)的直线方程为
($y_0$ - $y_1$)x + ($x_1$ - $x_0$)y + $x_0y_0$ = 0

3.点到直线的距离

点P(a, b)到直线f(x, y) = 0的有向距离d = $\frac{f(a, b)}{\sqrt{A^2 + B ^ 2}}$ 。
这里的有向距离是指，其绝对值等于距离值，但可正可负。在直线的一边距离是正的，而在另一边距离就是负的。尽管如此，但可以通过选择使之交换。方法是在f(x, y) = 0和-f(x, y) = 0中选择使用。

三角形及重心坐标系

1.重心坐标系

图形学中，经常希望为三角形的每个顶点赋以性质，如颜色，并将该性质的值插值到三角形中。实现这种要求有多钟方法，其中最简单的方法是利用 重心坐标系。
由点a、b、c组成的三角形。其坐标原点为a，从a到b和c的向量为基向量。于是任意一点p可表示为

p = a + β(b - a) + γ(c - a)

等价于

p = (1 - β - γ)a + βb + γc

经常定义一个新的变量α，以改善方程的对称性：

α = 1 - β - γ

于是得到方程

p(α, β, γ) = αa + βb + γc

重心坐标系的一大特点是, 对于由点a、b、c组成的三角形,当且仅当下列条件满足时,点p位于三角形内部:

0 < α < 1
0 < β < 1
0 < γ < 1

如果其中一个坐标为0,而其他两个坐标位于0和1之间,那么该点位于三角形边上。如果有两个为0，剩下一个为1，则该点位于顶点。

2.如何计算点(x, y)的β值?

β值是以点到(c - a)为基向量的直线的距离。b的β值是1，所以有

β = $\frac{f{ac}(x, y)}{f{ac}(x_b, y_b)}$ = $\frac{(y_a - y_c)x + (x_c - x_a)y + x_ay_c - x_cy_a}{(y_a - y_c)x_b + (x_c - x_a)y_b + x_ay_c - x_cy_a}$
类似可求γ。
α = 1 - β - γ

画直线

1.基于隐式方程绘制直线

首先有直线的隐式方程：
f(x, y) = ($y_0$ - $y_1$)x + ($x_1$ - $x_0$)y + $x_0y_0$ = 0
假设$x_0\leq y_0$， 如果不满足，交换点的顺序使之成立。直线的斜率为m=$\frac{y_1-y_0}{x_1-x_0}$。
在下面的讨论中都假设m ∈ (0, 1]。类似地，可以推出m ∈ (-$\infty$, -1]、m ∈ (-1, 0]和m ∈ (1, +$\infty$]时的结果。这4种情况覆盖了所有的可能情况。

绘制直线时，从左端点向右端点进行。只有两种可能：绘制一像素时，和左边所所绘制像素高度一致，或者高出一个像素。在两端点之间每一列，总是只有一个像素。没有像素就意味着有间隔，有两个像素则直线太粗。因为直线更接近水平，在水平方向的前进速度比垂直方向要快。有如下算法:
y = $y_0$
for x = $x_0$ to $x_1$ do
  draw(x, y)
  if(some condition) then
    y = y + 1
从左至右不断绘制像素，在该过程中有时需要向y轴正向有所偏移。关键是如何确定if语句中的条件。
一种有效方法是，参考两候选像素点之间的中点位置。对于刚绘制的像素点(x, y),它在屏幕上的坐标为(x, y),下面要绘制的候选像素为(x+1,y)和(x+1,y+1)。两候选像点之间的中点为(x+1, y+0.5)。如果直线通过中点的下方，就绘制下面的像素，否则绘制上面的像素，如图：

判断直线是通过点(x+1, y+0.5)的上方还是下方,可以通过式f(x+1, y+0.5)的值来判断。但很难利用f(x,y)的正负来快速判断是位于直线的上、下哪个方向。但是，可以通过分析算出来。($x_1-x_0$)肯定为正数，这就意味着随着y的增大，($x_1-x_0$)y项的值将变大。因此f(x, +$\infty$)肯定为正，并且位于直线的上方，也就是说直线上方的点都是正的。
于是，我们可以把if语句中的条件写具体：

if f(x+1, y+0.5) < 0 then

还有一种更高效率,就是 增量方法。以后再讨论。

有一过点$p_0=(x_0,y_0)$、$p_1=(x_1,y_1)$、$p_2=(x_2,y_2)$的三角形，顶点颜色分别为$c_0, c_1, c_2$。在重心坐标为(α,β,γ)的三角形内部，某点的颜色值为
$c = αc_0 + βc_1 + γc_2$

蛮力光栅化算法如下:
for all x do
 for all y do
  compute (α,β,γ) for (x,y)
  if (α∈[0,1] and β∈[0,1] and γ∈[0,1]) then
   $c = αc_0 + βc_1 + γc_2$
   drawpixel (x,y) with color c

寻找三顶点的包围矩形,只对该矩形内的候选像素执行循环。利用上面的公式计算重心坐标，算法变为：
$x{min} = floor(x_i)$
$x{max} = ceiling(xi)$
$y{min} = floor(yi)$
$y{max} = ceiling(yi)$
for $y = y{min}$ to $y{max}$ do
 for $x = x{min}$ to $x{max}$ do
  α = $\frac{f{12}(x,y)}{f{12}(x_0,y_0)}$
  β = $\frac{f{20}(x,y)}{f_{20}(x_1,y_1)}$
  γ = 1 - α - β
  if (α > 0 and β > 0 and γ > 0) then
   $c = αc_0 + βc_1 + γc_2$
   drawpixel (x,y) with color c

排序

发表于 2019-10-04 更新于 2020-02-15

说在之前

为什么要从排序说起呢？虽然各种语言提供了排序的接口，比如C#的List.Sort()，lua的table.sort()，但如果知道排序具体的实现，对学习和了解其他算法有更好的帮助。
在此之前，先给出排序的模板代码和测试代码。

public class Example
{
    public static void Sort(Comparable[] a)
    {
        //todo
    }

    private static bool Less(Comparable a, Comparable b)
    {
        return a.CompareTo(b) < 0 ;
    }

    private static void Swap(Comparable[] a, int i, int j)
    {
        Comparable tmp = a[i];
        a[i] = a[j];
        a[j] = tmp;
    }
}

对于每种排序，我们需要计算

  1. 运行时间：比较和交换的次数。对于不交换元素的算法，我们要计算访问数组的次数
  2. 额外的内存使用
  3. 稳定性

什么是稳定性？
  如果一个排序算法能够保留数组中重复元素的相对位置则可以被称为是稳定的。

初级排序算法

①选择排序

首先，找到所有元素中最小的一个，和第一个交换。在剩下的元素中找到最小的和第二个交换。以此往复，直到最后一个元素。

public class Selection
{
    public static void Sort(int[] a)
    {
        for(int i = 0, len = a.Length; i < len; i++)
        {
            int min = i;
            for(int j = i + 1; j < len; j++)
            {
                if(Less(a[j], a[min])) 
                    min = j;
            }
            Swap(a, i, min);
        }
    } 
    ...
}

从代码可以看出：对于长度为N的数组，总共经过 (N-1)+(N-2)+…+2+1 = $\frac{N(N-1)}{2}$ $\approx$ $\frac{N^2}{2}$次交换。交换元素在内循环之外，且每次都能排定一个元素，总交换次数是N，所以算法的时间效率取决于比较的次数。不需要额外空间。
它有两个很鲜明的特点：

  1. 运行时间和输入无关。白话就是两个相同长度的数组，一个已经排序好，一个随机排列，它们用的时间是一样的。即最优和最坏时间复杂度都是o($n^2$)
  2. 数据移动次数最少。这是下面任何算法都不具备的。

②插入排序

想一想我们平时打斗地主或是跑得快，牌堆是乱序的。对于每个人，从抓牌开始到结束，其实就是插入排序的过程。每次抓牌前，手牌都已经是从小到大排序好的；抓牌后，把它插入到合适的位置保证手牌是有序的。

public class Insertion
{
    public static void Sort(int[] a)
    {
        for(int i = 1, len = a.Length; i < len; i++)
        {
            for(int j = i; j > 0 && Less(a[j], a[j-1]); j--)
            {
                Swap(a, j, j-1)
            }
        }
    } 
    ...
}

对于随机排列长度为N且主键不重复的数组，平均情况下需要$\frac{N^2}{4}$次比较以及$\frac{N^2}{4}$次交换。最坏情况下需要$\frac{N^2}{2}$次比较以及$\frac{N^2}{2}$次交换。最好情况下需要N-1次比较和0次交换。

倒置 指的是数组中的两个顺序颠倒的元素。
假如我们要将数组[3,4,5,2,1]排序，它有7对倒置：3-2、3-1、4-2、4-1、5-2、5-1、2-1。

插入排序需要的交换次数和倒置的数量相同，这个很好理解。当数组中倒置数量为0时，数组有序。
每次交换都伴随着一次比较。当每个元素达到它最终的位置且不是第一个元素时，它还会和它前面的元素比较一次。所以 $倒置数量\leq比较次数\leq(倒置数量+N-1)$
根据以上性质，当倒置的数量很少时，插入排序很可能比本文其他算法都要快。

④希尔排序

对于大规模乱序数组插入排序很慢，因为它只会交换相邻的元素，因此元素只能一点一点地从数组的一端移动到另一端。希尔排序的思想是使数组中任意间隔为h的元素都是有序的。这样的数组被称为 h有序数组。一个h有序数组是h个互相独立的有序数组组成在一起的。

如果h很大，我们就能将元素移动到很远的地方，为实现更小的h有序创造方便。下面的例子使用了从$\frac{N}{3}$递减至1的序列。这个序列称为 递增序列 。下面的代码是实时计算的，另一种方式是提前计算，将这个序列存储在一个数组中。

public class Shell
{
    public static void Sort(int[] a)
    {
        int len = a.Length;
        int h = 1;
        while(h < len/3)
            h = 3 * h + 1;  //1, 4, 13, 40, 120, 364, 1093, ...

        while(h >= 1)
        {
            for(int i = h; i < len; i++)
            {
                for(int j = i; j >= h && Less(a[j], a[j-h]); j-=h)
                    Swap(a, j, j-h);
            }
            h/=3;
        }
    } 
    ...
}

希尔排序的轨迹如下图所示。

希尔排序可以看成是多次的插入排序，只不过前面的插入排序会为后面的插入排序提供便利。最好情况下需要常数次的比较和0次交换。最坏情况下需要$N^{1.5}$次比较。不需要使用额外的内存空间。

我该使用什么排序？

选择插入冒泡的平均复杂度都在平方级别，其他复杂算法除开希尔都是对数级别。

  1. 当集合中只有很少的元素不在正确的位置时可以使用插入排序。
  2. 快速排序是最快的通用排序算法。
  3. 如果稳定性是必要的，那归并排序可能是最好的。

大部分高级语言自带的排序接口，当元素是基本数据类型时使用快速排序，否则使用归并排序。
